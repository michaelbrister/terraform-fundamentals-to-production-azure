# Azure Course Progression Checklist

## Orientation

- [ ] Read Azure edition overview
- [ ] Confirm Terraform and Docker or local `miniblue` install are ready
- [ ] Start `miniblue` successfully
- [ ] Confirm local Azure endpoints respond
- [ ] Understand naming, region, and tagging conventions

## Associate track (01-13)

- [ ] 01 Local basics with Azure resources
- [ ] 02 Variables, locals, and outputs
- [ ] 03 `for_each` patterns
- [ ] 04 Dynamic blocks and optional config
- [ ] 05 Backend bootstrap concepts with Azure Storage backend design
- [ ] 06 State migration concepts and `azurerm` backend workflow
- [ ] 07 Modules
- [ ] 08 Console drills
- [ ] 09 Refactor without replacement
- [ ] 10 Break/fix troubleshooting
- [ ] 11 Tests and CI mindset
- [ ] 12 Timed exam drills
- [ ] 13 State subcommands

## Team and Governance track (14-16)

- [ ] 14 Workspaces and runs
- [ ] 15 RBAC and variable governance
- [ ] 16 Policy and approvals

## Professional track (17-22)

- [ ] 17 Import and adopt
- [ ] 18 Module versioning and promotion
- [ ] 19 Multi-team boundaries
- [ ] 20 Policy hardening
- [ ] 21 Incident recovery
- [ ] 22 Capstone

## Azure extension track (23-25)

- [ ] 23 HTTP API and Function App
- [ ] 24 Entra auth
- [ ] 25 Event Grid to Function

## Graduation criteria

- [ ] Dev, stage, and prod examples are reproducible
- [ ] Learner can explain why Labs `05` and `06` are concept-first instead of fully local remote-state labs
- [ ] CI is green and policy examples are understood
- [ ] Capstone documentation is complete
- [ ] Learner can explain backend, auth, emulator limits, and governance choices clearly
